HTML

HTML5中的语义化标签

header、nav、footer、article、section、aside

对语义化的理解

在没有CSS的情况下，页面也能呈现出很好的内容结构、代码结构。需要选择恰当的语义化标签，便于浏览器的爬虫解析，便于开发者维护

CSS

em和rem

em相对于父元素，rem相对于根元素

animation 和 transition什么区别?

transition强调过渡，需要事件触发，animation帧动画，不需要触发事件通过@keyframe也能随时间变化达到动画效果

三栏布局

两边固定，中间自适应

方法 具体操作 优缺点
浮动 左右浮动，中间自适应 要做清除浮动，可能高度塌陷
绝对定位 中间设置左右定位等于左右的宽度 脱离文档流
flex display：flex，左右设置宽度，中间flex：1自适应 完美但是不兼容IE8以下
表格布局 display: table-cell 左右定宽即可
网格布局 display: grid 具体没接触使用过

JS

ajax相关：

  1. 步骤：new XMLHttpRequest、open、send、onreadystatechange（判断readyState）
  2. readyState状态：0、1、2、3、4、5

    promise

    一种实现异步更好的方式，通过.then、.catch使得执行代码和处理结果的代码分离，符合人的逻辑思维习惯，避免回调地狱

    数据类型相关：

  3. js中有的数据类型：简单类型undefined、null、number、string、Boolean，复杂数据类型object
  4. typeof返回值：undefined、Boolean、string、number、function、object
  5. 判断变量是否是数组类型：instanceof、Array.isArray、判断变量.constructor是否是Array、Object.prototype.toString.call(arr)

框架

mvc，mvp和mvvm的区别

  1. MVC：Model（和数据库、数据有关的操作）-View（用户界面）-Control（业务逻辑），Controller 非常薄，只起到路由的作用，而 View 非常厚，业务逻辑都部署在 View
  2. MVP：Model-Presenter-Control，与MVC唯一的差别是Model和View之间不进行通讯，都是通过Presenter完成，View 非常薄，不部署任何业务逻辑，而 Presenter非常厚，所有逻辑都部署在那里
  3. MVVM：Model-View-ViewModel，数据和UI双向绑定，View的变动，自动反映在 ViewModel，反之亦然，减少DOM操作

计算机网络

restful

RESTful的核心就是后端将资源发布为URI，前端通过URI访问资源，并通过HTTP动词表示要对资源进行的操作。

http状态码：

  • 1xx：相关信息
  • 2xx：操作成功，200OK
  • 3xx：重定向，301永久重定向，302临时重定向，304客户端有缓存情况下服务端的响应
  • 4xx：客户端错误，400客户端请求的语法错误，404请求的资源不存在
  • 5xx：服务器错误，500服务器错误

    put

    最后一次修改后的资源，总是和第一次修改后的一样
